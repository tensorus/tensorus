# Contributing to Tensorus

Thank you for your interest in contributing to Tensorus! This guide covers contribution workflows for **both human and AI agent contributors**.

## ü§ñ AI Agent Participation: A Core Principle

Tensorus embraces AI agents as **essential contributors**. We believe the most innovative open source projects are built through human-AI collaboration. Whether you're a human developer, an AI coding assistant, or an autonomous agent‚Äîyour contributions are valued.

## üéØ Quick Start Guide

### For Human Contributors

1. **Fork the repository** on GitHub
2. **Clone your fork**: `git clone https://github.com/YOUR_USERNAME/tensorus.git`
3. **Create a feature branch**: `git checkout -b feature/your-feature-name`
4. **Make your changes** with tests and documentation
5. **Run the test suite**: `pytest tests/`
6. **Submit a pull request** with clear description

### For AI Agent Contributors

1. **Authenticate** via the Tensorus API or GitHub bot interface
2. **Declare your identity** as an AI agent in your first contribution
3. **Follow attribution standards** (see below)
4. **Generate comprehensive tests** alongside all code changes
5. **Submit PRs** via standard GitHub workflow or automated systems
6. **Include uncertainty markers** when appropriate

## üîÑ Development Workflow

### 1. Setting Up Your Environment

```bash
# Clone the repository
git clone https://github.com/tensorus/tensorus.git
cd tensorus

# Create virtual environment
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
pip install -r requirements-test.txt

# Install in development mode
pip install -e .
```

### 2. Making Changes

**Branch Naming Convention:**
- Feature: `feature/description`
- Bug fix: `fix/issue-number-description`
- Documentation: `docs/description`
- Performance: `perf/description`
- AI contribution: `ai/model-name/description`

**Commit Message Format:**
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types:**
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation only
- `perf`: Performance improvement
- `test`: Adding tests
- `refactor`: Code refactoring
- `ai`: AI-generated contribution

**Example:**
```
feat(tensor_ops): Add quantum-inspired decomposition algorithm

Implements tensor train decomposition with adaptive rank selection.
Performance tested on large-scale tensors (up to 10GB).

Generated-by: Claude 3.5 Sonnet
Reviewed-by: @maintainer_name
Closes #123
```

### 3. Testing Your Changes

```bash
# Run all tests
pytest tests/

# Run specific test file
pytest tests/test_tensor_ops.py

# Run with coverage report
pytest --cov=tensorus --cov-report=html tests/

# Run performance benchmarks
python benchmarks/run_benchmarks.py
```

### 4. Code Quality Checks

```bash
# Format code (if using Black)
black tensorus/

# Type checking (if using MyPy)
mypy tensorus/

# Linting (if using Ruff)
ruff check tensorus/
```

## ü§ñ AI Agent Specific Guidelines

### Autonomous AI Agents

If you're an autonomous AI agent making contributions:

1. **Register Your Agent**
   - Email: ai@tensorus.com
   - Include: Model name, capabilities, intended contribution scope
   - Receive: API access token and contribution guidelines

2. **Attribution Requirements**
   ```python
   # AGENT_INFO = {
   #     "model": "GPT-4 Turbo",
   #     "organization": "OpenAI",
   #     "task": "tensor_operations_optimization",
   #     "timestamp": "2025-01-14T10:30:00Z",
   #     "confidence": 0.95
   # }
   ```

3. **Quality Standards**
   - Generate comprehensive tests with >90% coverage
   - Include performance benchmarks for optimization claims
   - Add uncertainty markers for edge cases
   - Request human review for critical sections

4. **Safety Requirements**
   - No modifications to authentication/security systems without human approval
   - No automated merging of code
   - Rate limiting: Maximum 10 PRs per day
   - All PRs must pass CI/CD checks

### AI-Assisted Human Development

If you're a human using AI coding assistants:

1. **Transparency**: Mention significant AI assistance in PR description
2. **Verification**: Review and understand all AI-generated code
3. **Testing**: Add your own tests to validate AI output
4. **Attribution**: Optional but encouraged for substantial AI contributions

**Example:**
```markdown
## Development Notes
This PR was developed with assistance from GitHub Copilot for 
boilerplate code generation. All logic and algorithms were 
human-designed and verified.
```

**For All PRs:**
1. **Automated checks** must pass (tests, linting, type checking)

**Review Criteria:**
- Correctness and functionality
- Test coverage and quality
- Performance impact
- Security implications
- Documentation completeness
- Code style consistency
